# CostWiseAI 제출용 기획서

작성일: 2026-04-26  
문서 버전: v1.0

## 1. 프로젝트 개요

CostWiseAI는 보험/금융 조직에서 분리되어 운영되던 원가관리회계와 프로젝트 가치평가를 하나의 플랫폼으로 통합해, 의사결정 속도와 근거 추적성을 높이기 위한 서비스다.  
본 시스템은 본부별 원가 집계, 프로젝트별 수익성 및 리스크 평가, 승인/감사 이력 관리를 단일 흐름으로 연결한다.

## 2. 추진 배경 및 문제 정의

현업에서는 원가 자료, 투자평가 자료, 승인 이력이 각각 다른 채널에 흩어져 있어 다음 문제가 반복된다.

- 동일 프로젝트의 숫자가 부서/시점마다 다르게 해석됨
- 평가 결과는 있으나 입력 가정과 변경 이력을 역추적하기 어려움
- 승인/보류 판단 근거가 문서와 시스템에 분산되어 감사 대응이 느림

## 3. 목표

- 원가-평가-승인 데이터를 프로젝트 단위로 일관되게 연결
- 결과 수치뿐 아니라 산출 근거와 이력까지 확인 가능한 구조 구현
- 역할 기반 권한 통제로 운영 안정성과 감사 대응력 확보

## 4. 사용자 및 역할

| 역할 | 주요 책임 |
| --- | --- |
| ADMIN | 사용자/권한/운영 정책 관리, 전체 데이터 접근 |
| MANAGER | 프로젝트 운영, 원가/평가 검토 및 워크플로우 진행 |
| AUDITOR | 감사 로그 확인, 규정 준수 관점의 추적/검증 |

## 5. 핵심 기능 범위

1. 포트폴리오 및 프로젝트 요약 대시보드
2. 원가관리회계 요약 및 배부 근거 조회
3. 프로젝트 가치평가(DCF) 및 리스크 지표(VaR/Duration/Convexity) 조회
4. 승인 워크플로우(상태 전이, 리뷰 기록)
5. 감사 로그 등록/조회 및 사용자 관리

## 6. 적용 기술 및 운영 전략

- Frontend: React 18, TypeScript, Vite, Tailwind CSS
- Backend: Java 21, Spring Boot 3.x, Spring Security, Spring JDBC
- DB/Auth: Supabase PostgreSQL, Supabase Auth(JWT)
- 배포: Cloudflare Pages(프론트), Railway(백엔드), Supabase(DB/Auth)

운영 관점에서는 API 중심 구조를 기준으로 화면과 데이터 저장소를 일치시키고, 인증/인가 정책을 서버에서 강제한다.

## 7. 단계별 추진 계획

1. 1단계: 통합 데이터 기반 구축
- 프로젝트/시나리오/워크플로우/감사 로그를 DB 스키마 기준으로 정리
- 핵심 API와 화면을 연결해 데이터 단절 해소

2. 2단계: 추적성 강화
- 원가/평가 결과의 입력값, 계산 근거, 변경 이력 노출 강화
- 승인 흐름과 감사 기록의 연계 검증

3. 3단계: 운영 고도화
- 성능/모니터링/보안 설정 정교화
- 운영 자동화 및 문서 체계 표준화

## 8. 기대 효과

- 의사결정 리드타임 단축: 단일 대시보드에서 핵심 지표 동시 확인
- 설명 가능성 향상: 결과값과 근거 데이터의 연결 조회
- 감사 대응 개선: 역할/이력 기반 추적 체계로 증적 확보 용이

## 9. 완료 기준

- 프론트-백엔드-DB 간 프로젝트 핵심 수치 일관성 확보
- 권한별 접근 정책이 API에서 강제되고 테스트로 검증됨
- 승인/감사 이력이 운영 시나리오 기준으로 재현 가능함
