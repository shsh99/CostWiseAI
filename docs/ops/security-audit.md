# 보안 및 감사 흐름

## 목표

보험사/금융사 신규 사업 의사결정 플랫폼의 인증, 권한, 감사 흐름을 MVP 기준으로 정리한다.

## 현재 정책

- 개발/로컬 단계에서는 HTTP Basic을 유지한다.
- 운영 전환 시에는 Supabase JWT Bearer 토큰을 검증하는 방향으로 바꾼다.
- 프론트의 역할 표시는 참고 정보일 뿐, 권한 판단은 항상 백엔드가 수행한다.

## 역할 모델

- `planner`: 사업 초안과 가정값을 작성한다.
- `finance_reviewer`: ABC 배부와 DCF 가정을 검토한다.
- `executive`: 최종 승인/보류 결정을 내린다.

## 감사 이벤트

최소한 아래 작업은 이력으로 남긴다.

- 프로젝트 생성 및 수정
- 가정값 변경
- ABC 원가배분 재계산
- DCF 재평가
- 승인/반려 결정
- 권한 거부

## 최소 하드닝

- 서버는 stateless 세션으로 동작한다.
- form login과 logout redirect는 끈다.
- `OPTIONS`는 공개하고, 실제 API는 인증을 요구한다.

## 남은 작업

- JWT 인증 연동
- 역할별 권한 세분화
- 감사 이벤트 영속화
- 보안 헤더와 CORS 정책 구체화
