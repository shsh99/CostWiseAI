# 2026-04-21 Cockpit Readability

## 대상

- 브랜치: `feat/41-cockpit-readability`
- 범위: 이슈 `#41` cockpit 가독성/복잡도 개선

## 검증 기준(사전 고정)

- 첫 화면에서 핵심 KPI와 다음 액션이 즉시 식별되어야 한다.
- 탭별 콘텐츠가 과밀하지 않고 스캔 가능한 밀도를 유지해야 한다.
- 모바일/좁은 화면에서 레이아웃 겹침이 없어야 한다.
- 탭 접근성과 키보드 포커스 흐름은 유지되어야 한다.

## A/B 비교

### A. 기존 레이아웃 유지 + 텍스트 축약

- 장점: 변경 범위가 작다.
- 단점: 구조 자체가 복잡해 가독성 개선 효과가 제한적이다.

### B. 읽기 순서 중심 재배치 + 스캔 레일 추가

- 장점: 핵심 정보 우선순위를 시각적으로 강제해 빠른 판단이 가능하다.
- 단점: 스타일/레이아웃 수정 범위가 커진다.

## 선택

- `B`를 선택했다.
- 이유: 이슈 목표(가독성, 복잡도 감소, 스캔성 강화)에 직접 대응한다.

## 구현 요약

- summary/meta 이후 `cockpit-scan-rail`을 추가해 판단 흐름을 명시화
- 탭 내 패널 밀도와 텍스트 블록을 재정렬해 스캔 우선순위 강화
- 핵심 KPI/next action이 먼저 보이도록 배치
- 탭 접근성 시맨틱과 키보드 동작은 유지

## 검증

- `npm run lint` 통과
- `npm run build` 통과

## 남은 리스크

- next action 문구가 정적 매핑 기반이라 실제 데이터 기반 동적 추천으로의 확장 여지가 있다.
- 긴 텍스트 데이터에서는 소형 화면 줄바꿈이 늘어날 수 있어 문구 길이 가이드가 필요하다.
