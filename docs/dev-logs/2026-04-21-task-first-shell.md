# 2026-04-21 Task-First Shell Slice

## 대상

- 이슈: `#22`
- 브랜치: `feat/22-shell-workspace`
- 범위: 글로벌 셸/네비게이션 재구성

## A/B 비교

### A. 기존 단일 캔버스를 유지한 채 일부 카드와 탭만 재배치

- 장점: 변경량이 작고 현재 데모 흐름을 많이 유지한다.
- 단점: 포트폴리오, 프로젝트 상세, 역할 전환, 리뷰 레이어가 계속 같은 화면에서 경쟁한다.

### B. task-first 셸로 전환해 플랫폼/포트폴리오/워크스페이스 계층을 분리

- 장점: 글로벌 네비게이션과 로컬 워크스페이스를 구분할 수 있고 프로젝트 상세를 의도적 진입으로 바꿀 수 있다.
- 단점: `App.tsx`와 셸 CSS를 크게 재구성해야 한다.

## 선택

- `B`를 선택했다.
- 이유: 이 이슈의 수용 기준은 정보 구조와 진입 경로를 다시 세우는 것이며, 기존 단일 캔버스 유지안은 그 기준을 충족하지 못한다.

## 구현 요약

- 사이드바를 제품 맵 전용 글로벌 네비게이션으로 축소
- 탑바를 breadcrumb, 현재 페이지 설명, 역할/프로젝트 컨텍스트로 재구성
- 기본 랜딩을 dashboard/portfolio 중심 흐름으로 정리
- portfolio에서 project workspace로 의도적으로 진입하는 구조 추가
- management accounting / financial evaluation / reviews / settings를 별도 레벨로 분리
- 프로젝트 수준 로컬 네비게이션 원칙을 workspace 화면에 반영

## 검증

- `npm run build`
- `npm run lint`
- 결과: 모두 성공

## 남은 리스크

- 현재 구현은 `App.tsx` 중심의 셸 재구성이다. 이후 `#23`, `#24`에서 컴포넌트 분해가 필요하다.
- 좁은 뷰포트 반응형은 CSS 기준으로 보강했지만 실제 브라우저 수동 확인은 아직 하지 않았다.
