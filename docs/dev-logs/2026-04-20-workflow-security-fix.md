# 2026-04-20 워크플로우 및 보안 계약 정리

## 배경
- 승인 워크플로우와 감사 로그에 대한 테스트가 추가되면서, 인증 방식과 상태 전이 규칙을 명시적으로 맞출 필요가 생겼다.

## 수정한 내용
- `X-Auth-User` / `X-Auth-Role` 헤더를 인증 컨텍스트로 읽는 필터를 추가했다.
- `audit-logs`는 `EXECUTIVE` 역할만 접근하도록 정리했다.
- 리뷰 API는 헤더 기반 인증 정보를 사용하도록 바꿨다.
- `COMMENT` 액션은 상태를 유지한 채 감사 로그만 남기도록 정리했다.
- `IllegalArgumentException`은 공통 400 응답으로 변환되도록 했다.

## 검증
- `./gradlew.bat test` 통과
- `frontend`: `npm run lint` 통과
- `frontend`: `npm run build` 통과

## 남은 점검
- 현재 구현은 테스트 계약과 맞췄지만, 실제 운영용 인증 체계로 바꿀 때는 헤더 기반 인증을 별도 인증 계층으로 교체해야 한다.
